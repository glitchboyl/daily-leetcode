在歌曲列表中，第 i 首歌曲的持续时间为 time[i] 秒。

返回其总持续时间（以秒为单位）可被 60 整除的歌曲对的数量。形式上，我们希望索引的数字  i < j 且有 (time[i] + time[j]) % 60 == 0。

 

示例 1：

输入：[30,20,150,100,40]
输出：3
解释：这三对的总持续时间可被 60 整数：
(time[0] = 30, time[2] = 150): 总持续时间 180
(time[1] = 20, time[3] = 100): 总持续时间 120
(time[1] = 20, time[4] = 40): 总持续时间 60

示例 2：

输入：[60,60,60]
输出：3
解释：所有三对的总持续时间都是 120，可以被 60 整数。

 

提示：

  1. 1 <= time.length <= 60000
  2. 1 <= time[i] <= 500
解题思路:
第一种解法是暴力循环解法，直接在循环的当前项与后面的每一项相加的和整除 60 等于 0 的话计数加 1 即可。
这是最简单的解法，但是如果数组的长度超出一定范围可能就会超时。

所以正因为此我才去看了别人的第二种解法，来自作者lefter-2。
这种解法是通过 60 与当前数整除 60 的差的绝对值再整除 60，得到一个项数。在这个项数上加一，每次循环加上获取到这个项数的值。最后得出的和就是结果。
因为只要能整除 60 的数，加多少个 60 结果都是一样的。所以反过来说，(60 - 当前数) 整除 60 结果一样的数就是和能整除 60 的数。
